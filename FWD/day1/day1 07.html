Glossary on full stack web development

  1. Full Stack :

    2. A software development approach that includes both frontend and backend development.

    • In this example, the developer creates both the frontend and backend components of a web application. They may use HTML, CSS, and JavaScript for the frontend, and a programming language like Python or Ruby for the backend. The developer then integrates these components to create a functional web application.

  2. Frontend :

    3. The user-facing side of the application, including the interface, user interactions, and client-side processing.

  3. HTML :

    4. A markup language used to structure the content on the web.

    • In this example, the developer uses HTML to create the structure of a web page. They may create a header, navigation bar, main content area, and footer.

  4. CSS :

    5. A stylesheet language used to describe the look and formatting of a document written in HTML.

    • In this example, the developer uses CSS to style the web page created with HTML. They may define the colors, fonts, and layout for the various elements of the web page.

  5. Java Script :

    6. A scripting language that allows you to create dynamic and interactive websites.

    • In this example, the developer uses JavaScript to add interactivity to the web page. They may create a script that updates the content of a section of the web page or implements a feature that allows users to submit feedback.

  6. Backend :

    7. The server-side of the application, including database management, business logic, and server-side processing.

  7. React:

    8. A popular JavaScript library for building user interfaces, particularly single-page applications.

    • In this example, the developer uses React to create the frontend of a web application. They may use React components to build the structure and appearance of the web page, and incorporate React's state management features to add interactivity to the web page.

  8. Node.js:

    9. An open-source, cross-platform runtime environment that allows JavaScript to be executed on the server-side.

    • In this example, the developer uses Node.js to run a JavaScript script on the server-side. They may use Node.js to create a backend application that interacts with a database to retrieve and store data.

  9. Angular:

    10. A powerful, modern web framework used to build dynamic, interactive, and high-performance web applications.

    • In this example, the developer uses Angular to create a full-featured, single-page web application. They may use Angular components to build the structure and appearance of the web page, and Angular services to fetch and manage data.

  10. Database :

    11. A structured set of data held in a computer or other electronic device, consisting of fields, records, and often indices or keys.

    • In this example, the developer uses a database to store and manage the data of their web application. They may create tables to define the structure of the data, and they may use SQL queries to perform operations on the data stored in the database.

  11. API

    12. Application Programming Interface. A set of rules and protocols for building software applications. An API defines the methods and data formats that applications can use to request and exchange information.

    • In this example, the developer creates an API that allows other applications to access and manipulate the data and functionality of their web application. They may use API endpoints to define the routes and methods that can be accessed, and they may implement authentication and authorization mechanisms to secure access to the API.

  12. Ruby on Rails:

    13. A server-side web application framework written in Ruby. It is designed to make programming web applications easier by making assumptions about the required features.

    • In this example, the developer uses Ruby on Rails to create a web application that follows the Model-View-Controller (MVC) design pattern. They may use Ruby on Rails' built-in features and tools to quickly build a fully functional web application.

  13. AWS:

    14. Amazon Web Services. A collection of cloud computing services offered by Amazon.

    • In this example, the developer uses Amazon Web Services (AWS) to deploy and host their web application. They may use AWS services like EC2 for virtual servers, S3 for cloud storage, and RDS for managed relational databases to create a scalable and secure hosting environment for their webÂ application.

  14. Git :

    15. A widely-used version control system for tracking changes in source code during software development.

    • In this example, the developer uses Git to track changes to their source code and collaborate with other developers. They may use Git commands like git init, git add, git commit, and git push to initialize a repository, add files, commit changes, and push changes to a remote repository.

  15. Django:

    16. A high-level Python web framework that encourages rapid development and clean, pragmatic design.

    • In this example, the developer uses Django to create a web application that follows the Model-View-Controller (MVC) design pattern. They may use Django's built-in features and tools to quickly build a fully functional web application.
